# B2B — движок удержания для СТО/дилеров (Dev 1)

Продаём автосервисам/дилерам: их клиенты и авто **уже в Bitrix24 CRM**. servys знает, когда у клиента
подходит ТО, и **сам создаёт дело-напоминание в CRM** («связаться с Ивановым — Camry, пора на ТО») →
знание о ТО превращается в поток обращений. Без OAuth — на **входящем вебхуке**.

## Минимальные b2b-сценарии

- **B2B-1 Подключение СТО.** СТО отдаёт входящий вебхук Bitrix24 → мы валидируем и храним (шифровано).
  `POST /api/v1/b2b/service-centers {name, webhook, responsible_id}`.
- **B2B-2 Чтение автопарка из CRM.** Читаем контакты клиентов + поля авто (марка/модель/год/пробег)
  через `crm.contact.list` (маппинг полей — `bitrix.ContactFieldMap`, дефолт `UF_CRM_CAR_*`).
- **B2B-3 Выявление ТО.** По каждому авто — тот же **рекомендательный слой** (`recommender.Advisor`),
  что и в b2c. Берём статусы SOON/DUE/OVERDUE.
- **B2B-4 Ретеншн-действие в CRM.** На подошедшее ТО создаём дело на контакте клиента
  (`crm.activity.todo.add`, ownerType=контакт) с дедлайном и ответственным. **Идемпотентно** (дедуп
  по `tenant|авто|правило|срок` в `pushed_actions`) — повторный скан не плодит дубли.
- **B2B-5 Скан.** `POST /api/v1/b2b/service-centers/{id}/scan` прогоняет B2B-2…B2B-4 и возвращает отчёт
  `{cars, due_items, pushed, skipped, errors}`. (Периодический шедулер — следующий шаг.)

## Архитектура (что где)

```
api/b2b.go        эндпоинты connect/list/scan (Dev 1)
b2b/              оркестратор ScanAndPush + порты FleetSource/Retention/DedupeStore (Dev 1)
bitrix/           CRMFleet (crm.contact.list) + CRMRetention (crm.activity.todo.add) — реализации портов
store/            service_centers (вебхук шифрован), pushed_actions (дедуп)
crypto/           AES-256-GCM для вебхуков (ключ APP_SECRET_KEY)
recommender.Advisor   переиспользуется из b2c (Dev 3) — ядро одно
```

## Включение

b2b активен только при `APP_SECRET_KEY` (иначе эндпоинты отдают 503). Live-прогон на реальном портале —
отдельно (см. `backend/bitrix/README.md`), здесь всё покрыто тестами против фейкового Bitrix.

## Осознанно НЕ в этой итерации

OAuth/Marketplace, мультиарендная авторизация b2b-эндпоинтов, периодический шедулер, обратная
синхронизация статусов из CRM, смарт-процессы/сделки вместо дел, отмена/закрытие дел.
