# Bitrix-коннектор (Dev 1)

Интеграция с Bitrix24 через **входящий вебхук** (без OAuth), за портом `sink.Sink`.
Создаёт задачу (`tasks.task.add`) по напоминанию. Этап **b2b** — в b2c-сборке не подключается.

## Живой прогон на своём портале

1. В Bitrix24: **Разработчикам → Другое → Входящий вебхук**, права `task`, `basic/profile`.
2. Скопируй URL вида `https://<портал>.bitrix24.ru/rest/<id>/<token>/`.
3. Запусти:

```bash
BITRIX_WEBHOOK_URL='https://<портал>.bitrix24.ru/rest/<id>/<token>/' \
  go test ./bitrix/ -run Live -v
```

В портале появится тестовая задача. Без `BITRIX_WEBHOOK_URL` тест пропускается,
а обычные юнит-тесты (валидация, отправка, ошибки) гоняются против фейкового эндпоинта.

## Безопасность

- Вебхук валидируется: только `https`, структура `/rest/{id}/{token}/`, без IP/localhost/query/userinfo.
- Токен **никогда не логируется** (маскируется в ошибках), таймаут 5с, тело ответа ограничено.

## Использование в коде

```go
s, err := bitrix.NewSink(webhookURL, responsibleID) // реализует sink.Sink
_ = s.Deliver(ctx, sink.Reminder{Vehicle: v, Alert: a})
```

Мультиарендность/хранение вебхуков per-tenant, шедулер-рассылка, CRM-сущности — **дальнейший b2b**, здесь не реализованы.
